# Evidence: https://web.stanford.edu/~ouster/cgi-bin/papers/OngaroPhD.pdf

> "Replicated state machines are typically implemented using a replicated log [...] Each server stores a log containing a series of commands, which its state machine executes in order. At any given time each server is in one of three states: leader, follower, or candidate."

*Paraphrase:* The thesis develops the full Raft treatment cited by the axioms in `supports`: terms as a logical clock, leader election with randomized timeouts, log replication and the AppendEntries consistency check, the commit rule (current-term majority, never counting prior-term replicas), the four safety properties and the proof-by-contradiction for Leader/State-Machine Safety, the read-index and lease read-only optimizations, leadership transfer, Pre-Vote and CheckQuorum, single-server and joint-consensus membership changes, snapshotting and copy-on-write, persistence, and the comparison to Paxos/Viewstamped-Replication/Zab. Full thesis at the source URL above.
