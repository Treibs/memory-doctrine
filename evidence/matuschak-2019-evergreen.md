# Matuschak 2019 — Evergreen Notes (Atomicity)

Andy Matuschak's evergreen notes canon establishes that a note should be atomic —
focused on a single discrete idea — because atomicity is the precondition for
composability: only an atom can be linked precisely, reused across contexts, and
addressed as a stable unit across different arguments.

Matuschak explicitly states the rationale is **composability**, not retrieval: the
system accumulates generative power because atomic nodes can be recombined in
novel ways without carrying unwanted context from their origin. A multi-claim note
cannot be linked precisely; its connections are blurry and context-dependent.

Key passages used by [[A2-atomicity]]:
- "Evergreen notes should be atomic" — each note focuses on a single discrete idea
  "for easier retrieval and reuse."
- The companion note "Evergreen notes should be concept-oriented" extends the same
  logic: organize around concepts, not sources, so nodes are genuinely reusable
  across contexts.

The v1.1 spine correction is reflected here: the atomicity rationale is
**composability** (the node is reusable precisely because it carries one clean
idea), not retrieval speed per se — the 2026-06-03 purge caught and corrected
the earlier "retrieval" framing.
